# FixNow Project Overview

## 1) Project Summary
**FixNow** is a platform that connects users who need repair/maintenance support with suitable providers based on location and service category. The product aims to make the process of finding providers, booking support, and tracking progress faster, more transparent, and more reliable.

## 2) Problem Statement
In real-life scenarios, when household devices or infrastructure break down, users often face the following issues:
- Difficulty finding trustworthy providers quickly.
- Limited transparency in pricing, quality, and service progress.
- Fragmented communication flow (manual calls/messages), making request history hard to track.

FixNow addresses this with a centralized flow: create request → match provider → track status → keep history.

## 3) Product Goals
- Reduce time from issue occurrence to provider assignment.
- Improve service transparency (description, status, and feedback).
- Deliver a consistent mobile experience for both end users and operations teams.

## 4) Target Users
### Primary Users
- Individuals/households needing fast repair support (electricity, plumbing, appliances, etc.).

### Secondary Users
- Providers/service workers who want to receive jobs based on their expertise.
- Admin/operations staff monitoring service quality and system performance.

## 5) Core Value Proposition
- **Fast**: simple request creation and quicker access to suitable providers.
- **Reliable**: clear processing history and status visibility.
- **Convenient**: centralized request management in one app.

## 6) MVP Scope (Current Phase)
- User authentication (sign in/sign up, forgot password).
- Basic note/request creation and management.
- List view and search capabilities.
- User profile and activity view.

## 7) Out of MVP Scope
- Online payments.
- Full provider payroll/settlement by completed jobs.
- Advanced AI-based recommendation algorithms.

## 8) Suggested Success Metrics (KPIs)
- User completion rate for sign-up and sign-in.
- Average time from request creation to first response.
- Weekly/monthly returning user rate.
- Post-request satisfaction score (rating/feedback).

## 9) Product Direction
- Integrate production backend for authentication and request workflows.
- Add status-based notifications.
- Improve service quality review/rating mechanisms.
- Expand service categories and supported areas.

## 10) User Roles

The target operating model supports three primary roles. In the current MVP, these roles are defined at the business level and will be implemented incrementally.

### Customer
Users who create repair or maintenance requests.

Capabilities:
- Create service requests
- Upload photos/videos
- Track service progress
- Communicate with providers
- Provide ratings and feedback

### Provider
Workers who accept and complete service requests.

Capabilities:
- View available requests
- Accept or reject jobs
- Update work progress
- Upload completion proof
- Manage earnings

### Administrator
Responsible for platform management and monitoring.

Capabilities:
- Verify provider accounts
- Manage users
- Monitor service quality
- Handle disputes and complaints
